MongoDB入门

MongoDB

MongoDB 简介

MongoDB 是一个面向文档的数据库，mongodb中有三元素：数据库，集合，文档，其中“集合”，不再有预定义的模式(实际开发过程中还是会先确定业务模式)，可以很容易的添加或者删除字段。

就是对应关系数据库中的“表”，“文档”对应“行”。

mac 上安装MongoDB

MongoDB基本操作

  • 启动数据库
  • 展示所有的数据库、选择test库、展示所有的集合
  • 插入数据到新的集合(insert)

  • 查询刚才插入的数据(find)
    我们将数据插入后，肯定是要find出来，这里要注意两点：

    ① “_id”： 这个字段是数据库默认给我们加的GUID，目的就是保证数据的唯一性。

    ② 严格的按照Bson(json 的扩展)的形式书写文档

  • 更新操作，update接收至少两个参数：第一个是限定条件，第二个是更新的文档
  • 删除操作 remove

数据类型

基本数据类型

  • null
  • 数值 {‘x’: 3.14}(默认64位浮点型)
  • 字符串{‘x’:’hello’}
  • 布尔型{‘x’:true}
  • 日前 {‘x’: new Date()}
  • 正则表达式{‘x’: /foobar/i}
  • 数组 {‘x’: [‘a’,’b’,’c’]}
  • 内嵌文档 {‘x’: {‘foo’: ‘u’}}
  • 对象id {‘x’:ObjectId()}
    MongoDB中存储的文档必须有一个_id。这个键的值可以是任何类型，默认为ObjectId 对象

查询

指定需要返回的键

有时候，我们并不需要查看文档中所有的键/值，比如我们只想看集合中所有文档的age，那么只需要在find的第二个参数中指定需要返回的键即可。

默认情况下_id总是被返回，如果不想返回_id，也可以剔除

查询条件

  • $gt”, “$gte”, “$lt”, “$lte”, “$ne” 比较操作符 对应 >,>=,<,<=,!

OR 查询

MongoDB 中有两种方式可以进行OR 查询：

  • $in/$nin

  • $or

$not

$not 是元条件句，即可以用在任何其他条件之上。

特定类型的查询

null

  • 可以匹配数值为null的文档
  • 还会匹配不包含这个键的文档

如果只想匹配值为null，而不是没有这个键的记录那么需要使用$exists

正则表达式

MongoDB使用Perl兼容的正则表达式（PCRE)库来匹配正则表达式，任何PCRE支持的正则表达式语法都能被MongoDB接收，建议在查询中使用正则表达式前现在JavaScript shell 中检查下语法，确保结果和设想一致

查询数组

查询数组元素和查询标量值是一样的。

$all

如果需要通过多个元素来匹配数组，就要使用$all.

如果要匹配数组特定位置的元素，需要使用key.index语法指定下标:

数组下标从0开始

$size

可以用$size查询特定长度的数组

$slice

$slice 可以返回某个键匹配的数组元素的一个子集

只返回前2种水果

$where

查询中的无敌大招，可以写js语句进行查询返回，因为它几乎可以做任何事情，所以应该严格限制或者消除使用$where
