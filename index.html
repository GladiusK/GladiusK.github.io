Elasticsearch入门

Elasticsearch

简介

Elasticsearch是一个基于Apache Lucene(TM)的开源搜索引擎。无论在开源还是专有领域，Lucene可以被认为是迄今为止最先进、性能最好的、功能最全的搜索引擎库。
但是，Lucene只是一个库。想要使用它，你必须使用Java来作为开发语言并将其直接集成到你的应用中，更糟糕的是，Lucene非常复杂，你需要深入了解检索的相关知识来理解它是如何工作的。
Elasticsearch也使用Java开发并使用Lucene作为其核心来实现所有索引和搜索的功能，但是它的目的是通过简单的RESTful API来隐藏Lucene的复杂性，从而让全文搜索变得简单。

Linux下安装Elasticsearch2.2.0

说明：需要依赖jdk,jdk和elasticsearch 版本关系

  • curl -L -O https://download.elastic.co/elasticsearch/release/org/elasticsearch/distribution/tar/elasticsearch/2.2.0/elasticsearch-2.2.0.tar.gz
  • tar -xvf elasticsearch-2.2.0.tar.gz
  • cd elasticsearch-2.2.0/bin
  • ./elasticsearch

启动后所有都是按照默认的配置，如果需要修改配置可以在/elasticsearch/config/ 中修改elasticsearch.ym

如果需要后台启动bin/elasticsearch -d -p [文件夹名称]

集群

一个节点(node)就是一个elasticsearch实例，一个集群(cluster)就是由一个或者多个节点组成，它们具有相同的cluster.name。默认的名字为’elasticsearch’,请注意区分各个不同环境中的集群名称以免发生节点的错误归并。

节点

一个节点(node)就是一个elasticsearch实例，是一个集群的一部分，用于存储数据并参与集群的索引和搜索功能，节点和集群一样是通过名称进行区分的，默认情况下使用Marvel名字。

Index(索引)

一个index 是一个含有类似特性documents的集合，比如：用户数据等，文档也是通过文档名字进行区分。

Type(类型)

在一个index内，你可以定义一个或者多个type,一个type是index的一个逻辑分类/分区，通常来说一个type里面包含一些拥有相同字段的documents。

Document(文档)

document是一个可以被索引的基础单元，使用JSON格式进行存储。

三者关系

在Elasticsearch中，文档归属于一种类型(type),而这些类型存在于索引(index)中，我们可以画一些简单的对比图来类比传统关系型数据库：
Relational DB -> Databases -> Tables -> Rows -> Columns
Elasticsearch -> Indices   -> Types  -> Documents -> Fields
Elasticsearch集群可以包含多个索引(indices)（数据库），每一个索引可以包含多个类型(types)（表），每一个类型包含多个文档(documents)（行），然后每个文档包含多个字段(Fields)（列）

主分片&复制分片

一个index可以存储TB级别的数据，但是如果将这么大的数据存储在一个node或者一个disk上将导致查询服务非常慢，而且也不利于数据的备份和处理。因此Elasticsearch提供分片技术，使得我们可以划分我们的海量index数据。每一个分片都是一个独立完整的index，能够对外提供服务。
分片的好处：

  • 它可以让你水平分割/扩展您的内容量

  • 它可以让你分布和并行跨分片操作（可能在多个节点），从而提高性能/吞吐量

    复制分片只是主分片的一个副本，它可以防止硬件故障导致的数据丢失，同时可以提供读请求，比如搜索或者从别的shard取回文档。
    默认情况下，elasticsearch中的每个index分片5个主分片，每个分片都有一个副本分片，当索引创建完成的时候，主分片的数量就固定了，但是复制分片的数量可以随时调整。
    一个分片(shard)是一个最小级别“工作单元(worker unit)”,它只是保存了索引中所有数据的一部分，分片就是一个Lucene实例，并且它本身就是一个完整的搜索引擎。我们的文档存储在分片中，并且在分片中被索引，但是我们的应用程序不会直接与它们通信，取而代之的是，直接与索引通信。

    查看集群状态

    curl ‘localhost:9200/_cat/health?v’

    列出所有index

    curl ‘localhost:9200/_cat/indices?v’

MongoDB入门

MongoDB

MongoDB 简介

MongoDB 是一个面向文档的数据库，mongodb中有三元素：数据库，集合，文档，其中“集合”，不再有预定义的模式(实际开发过程中还是会先确定业务模式)，可以很容易的添加或者删除字段。

就是对应关系数据库中的“表”，“文档”对应“行”。

mac 上安装MongoDB

MongoDB基本操作

  • 启动数据库
  • 展示所有的数据库、选择test库、展示所有的集合
  • 插入数据到新的集合(insert)

  • 查询刚才插入的数据(find)
    我们将数据插入后，肯定是要find出来，这里要注意两点：

    ① “_id”： 这个字段是数据库默认给我们加的GUID，目的就是保证数据的唯一性。

    ② 严格的按照Bson(json 的扩展)的形式书写文档

  • 更新操作，update接收至少两个参数：第一个是限定条件，第二个是更新的文档
  • 删除操作 remove

数据类型

基本数据类型

  • null
  • 数值 {‘x’: 3.14}(默认64位浮点型)
  • 字符串{‘x’:’hello’}
  • 布尔型{‘x’:true}
  • 日前 {‘x’: new Date()}
  • 正则表达式{‘x’: /foobar/i}
  • 数组 {‘x’: [‘a’,’b’,’c’]}
  • 内嵌文档 {‘x’: {‘foo’: ‘u’}}
  • 对象id {‘x’:ObjectId()}
    MongoDB中存储的文档必须有一个_id。这个键的值可以是任何类型，默认为ObjectId 对象

查询

指定需要返回的键

有时候，我们并不需要查看文档中所有的键/值，比如我们只想看集合中所有文档的age，那么只需要在find的第二个参数中指定需要返回的键即可。

$not

$not 是元条件句，即可以用在任何其他条件之上。

相关视频
在95%的动画组件使用情况中，我们不需要自己去使用贝赛尔曲线缓动和设置动画持续时间。设置UI元素的刚度和阻尼，然后的事就交给充满魔力的物理学。这样，你完全不用理会诸如动画行为被打断之类的大量问题。并且提供了一个非常易用的API

这个库同时为React 的 TransitionGroup 提供了一个更加好的api 选择

API

这个库对外提供Motion, StaggeredMotion, TransitionMotion, presets, spring 和 utils

spring: number -> ?[stiffness, damping] -> ConfigurationObject

(注意: 不包含 版本<0.3.0 的 Spring 组件)
一般用来辅助指定如何变化到目标值，比如 sprint(10,[120,17]) 返回一个包含私有属性的配置对象，其用来描述一个携带物理学刚度为120和阻尼为17’’如何变动到10。spring(10)不带数组参数时，其默认参数为[170,26].
下面会有更多的使用说明，库的配置参数详情点击

<Motion />

属性

defaultStyle: ?Object

可选的。接受一个任意键的对象，绑定值到你的动画的配置属性值中，这个值只在组件第一次渲染的时候起作用（之后的渲染会忽略这个属性）

style: Object

必填。在组件存在的整个周期内必须保持同样的键。如果提供了defaultStyle则必须保持一样的键。和defaultStyle类似，但是目标值需要查询spring的配置：{x: spring(10), y: spring(20, [120, 17])}
如果没有提供Spring配置，也没有使用下面的子函数参数，而是直接给定一个常数值，程序将直接变化到给定的值

children: Object -> ?ReactElement

需要 一个有 interpolated style 对象为参数的 function，比如：{x: 5.2, y: 12.1} ，必须返回一个React 元素进行渲染

<StaggeredMotion />

当你需要使一系列的元素动起来，你可以使用一个包含motion的数组，然后让数组中的元素挨个动起来。然而常见的情形是，我们想使他们间隔一段时间的一个接一个的动起来。硬编码动画持续时间似乎可以做到。但是，如果使用<StaggeredMotion />将会是一种更加自然的符合物理学的选择，可以让元素的目标位置基于另外一个元素的当前位置

defaultStyles: ?Array<Object>

可选.和Motion的defaultStyle属性类似

styles: ?Array<Object> -> Array<Object>

必填 function。将之前的styles数组作为函数的参数（如果没有提供defaultStyles则undefined作为第一次渲染时候的参数传递给函数

children: Array<Object> -> ?ReactElement

必填 funciton。类似Motion的children,但是接收一个 interpolated style 数组,比如 [{x:5},{x:8.1}]

<TransitionMotion />

一个可以帮助你增加和移除动画的神奇组件。不同于 React的TransitionGroup,TransitionMotion 用元素的styles对象属性的形式来做区分，而不是当元素消失的时候去保留一些元素。

普遍的思路
使用{myKey1: {x: spring(30)}, myKey2: {x: spring(20)}}作为TransitionMotion的styles。interpolated styles被传入到子函数中，一段时间后，参数变化为{myKey1: {x: 15.1}, myKey2: {x: 8.2}}

一段时间过后，你删除了mykey1 和它的style配置，因此新的的styles变为 {mykey2:{x:spring(20)}}。TransitionMotion发现了少了mykey1,但是在interpolated style中还是保留mykey1的值{myKey1: ..., myKey2: ...}。

当TransitionMotion调用你提供的willLeave 属性的时候，将mykey1作为参数传递。你需要在willLeave中为mykey1返回一个终态的style配置，比如：{x: spring(50)},作为其目标值，当interpolatedStyles.myKey1变化到这个值后，就可以让TransitionMotion从interpolated style真正的移除mykey1以及对应的style配置

终上所述:styles从初始状态 {k1: {x: spring(30)}, k2: {x: spring(20)}},再次渲染后styles为{k2: {x: spring(20)}}。传递给子函数的interpolated style 中的styles并没有被影响（还是包含了mykey1)，需要注意的是，在k1: configReturnedFromWillLeave({x: spring(50)})中返回的mykey1部分并没有真正的存在于styles中。一段时间过后，interpolated styles变为 {k1: {x: 50}, k2: {x: 19.2}}，然后再次渲染的时候，mykey1已经被删除，只剩下{k2: {x: 19.2}}。因此，在最后一次渲染之前，映射在interpolate styles的元素会渲染2次

defaultStyles: ?Object<string, Object>

可选。接受一个{myKey1: styleObject, myKey2: styleObject} 这样的对象，其中每一个styleObject的规则和Motion的defaultStyle一样。其中key必须为唯一的非数字

styles: Object | (?Object -> Object)

必填。接受一个类似defaultStyles的对象，但是styleObject需要有spring配置（{myKey1: {x: spring(10)}, myKey2: {y: spring(20)}}），或者接受一个带prevStyles 参数（类似StaggeredMotion，你可以交叉的取消动画）和返回值为目标style的方法

willEnter: (string, Object, Object, Object, Object) -> Object

可选。传递一个参数为(keyFromStylesThatJustEntered, correspondingStyleOfKey, styles, currentInterpolatedStyle, currentSpeed)返回值为类似defaultStyle的style对象的方法
方法的默认返回值为correspondingStyleOfKey，比如上面的例子为{x: spring(20)}。

willLeave: (string, Object, Object, Object, Object) -> Object

可选。传递一个参数为(keyThatJustLeft, correspondingStyleOfKey, styles, currentInterpolatedStyle, currentSpeed)并且返回值为包含spring(...)作为目标配置 的style对象

可选，默认为correspondingStyleOfKey，即立刻从interpolated style中移除key

children: Object -> ?ReactElement

必填 **function**.同Motion的children，不同的是接受一个interpolated styles对象

presets

你可以优雅的在你的style预置你的spring参数，就像这样spring(10, presets.wobbly). 点击详情.

utils

因为TransitionMotion规定styles必须为一个对象，操作对象中的key会比操作数组显得繁琐，下面有一些公共场景下的解决方案：

  • 在开始的时候插入元素:{newKey: myConfigForThisKey, ...oldConfigs}
  • 在最后的时候插入元素:{...oldConfigs, newKey: myConfigForThisKey}
  • Slice/splice/reverse/sort：utils.reorderKeys库提供这些方法
    注意：对象的keys的创建顺序默认为升序，TransitionMotion中不能将数字作为key。幸运的是，你可以增加一个字母将数字转换为字符。
    ###reorderKeys: (Object, Function) -> Object
    utils.reorderKeys({a: 1, b: 2}, (keysArray) => ['b', 'a']) // gives {b: 2, a: 1}

Function的参数为，在Object中的key数组并且返回一个新的keys数组（数组的顺序变化，或者新增/删除key）。reorderKeys最终返回一个由新的key排序方式的对象
